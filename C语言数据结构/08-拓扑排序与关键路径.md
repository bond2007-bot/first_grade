## 一、拓扑排序（Topological Sort）

**拓扑排序**：对有向无环图（DAG）的顶点进行排序，使得对每条弧 `<u, v>`，u 都在 v 之前。

### 1. 算法思想

```
1. 统计每个顶点的入度
2. 将所有入度为0的顶点入栈
3. 重复以下步骤直到栈空：
   ① 栈顶出栈并输出
   ② 删除以该顶点为起点的所有边（邻接点入度减1）
   ③ 若某个邻接点入度变为0，则入栈
4. 若输出的顶点数 < 总顶点数，说明图中存在环
```

### 2. 代码实现

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXVEX 20

typedef struct
{
    char vex[MAXVEX];          // 顶点表
    int arc[MAXVEX][MAXVEX];   // 邻接矩阵
    int vexnum;                // 顶点数
    int arcnum;                // 边数
}Graph;

void TopologicalSort(Graph G)
{
    int indegree[MAXVEX] = {0};  // 入度数组
    int stack[MAXVEX];           // 模拟栈
    int top = -1;

    int i, j, k;
    int count = 0;

    /* 计算各顶点入度 */
    for(i = 0; i < G.vexnum; i++)
    {
        for(j = 0; j < G.vexnum; j++)
        {
            if(G.arc[i][j] != 0)
                indegree[j]++;  // 有边i→j，则j的入度+1
        }
    }

    /* 入度为0的顶点入栈 */
    for(i = 0; i < G.vexnum; i++)
    {
        if(indegree[i] == 0)
            stack[++top] = i;
    }

    while(top != -1)  // 栈不为空
    {
        k = stack[top--];
        printf("%c ", G.vex[k]);  // 输出
        count++;

        /* 删除以k为起点的边 */
        for(j = 0; j < G.vexnum; j++)
        {
            if(G.arc[k][j] != 0)
            {
                indegree[j]--;  // 入度减1
                if(indegree[j] == 0)
                    stack[++top] = j;  // 入度为0则入栈
            }
        }
    }

    if(count < G.vexnum)
        printf("\n图中存在环，无法进行拓扑排序！\n");
}
```

### 3. 测试代码

```c
int main()
{
    Graph G;
    G.vexnum = 6;
    G.arcnum = 6;

    G.vex[0] = 'A'; G.vex[1] = 'B';
    G.vex[2] = 'C'; G.vex[3] = 'D';
    G.vex[4] = 'E'; G.vex[5] = 'F';

    // 初始化邻接矩阵
    for(int i = 0; i < G.vexnum; i++)
        for(int j = 0; j < G.vexnum; j++)
            G.arc[i][j] = 0;

    // 构造有向无环图
    G.arc[0][1] = 1;   // A→B
    G.arc[0][2] = 1;   // A→C
    G.arc[1][3] = 1;   // B→D
    G.arc[2][3] = 1;   // C→D
    G.arc[2][4] = 1;   // C→E
    G.arc[3][5] = 1;   // D→F

    printf("拓扑排序的结果是：");
    TopologicalSort(G);

    return 0;
}
```

```
图示：
    A ──→ B ──→ D ──→ F
     ↘   ↗     ↗
      C ──→ E

拓扑排序结果（不唯一）：A B C D E F  或  A C B D E F 等
```

### 4. 重要结论

- 拓扑排序结果**不唯一**
- 有回路的图**不能**进行拓扑排序
- 拓扑排序可以用**队列**或**栈**实现
- 时间复杂度：O(V + E)（邻接表）/ O(V²)（邻接矩阵）

## 二、关键路径（Critical Path）

### 1. 相关概念

| 概念 | 含义 | 说明 |
|------|------|------|
| **AOE网** | 边表示活动的网 | 顶点表示事件，边表示活动，权值表示活动持续时间 |
| **etv** | 事件最早发生时间 | 正推，取最大值 |
| **ltv** | 事件最晚发生时间 | 倒推，取最小值 |
| **关键路径** | etv和ltv相等的点构成的路径 | 决定整个工程的最短完成时间 |

### 2. 关键路径求解步骤

```
① 拓扑排序（求etv）
   正向计算事件最早发生时间
   etv[k] = max(etv[i] + weight<vi, vk>)

② 逆拓扑排序（求ltv）
   反向计算事件最晚发生时间
   ltv[i] = min(ltv[k] - weight<vi, vk>)

③ 计算活动的最早开始时间 e(i) 和最晚开始时间 l(i)
   e(i) = etv[活动起点]
   l(i) = ltv[活动终点] - 活动持续时间

④ 关键活动：e(i) = l(i) 的活动
   关键路径：关键活动组成的路径
```

### 3. 关键路径的应用

```
AOE网示例：
    开始 → A(3) → ③ → B(2) → 结束
      ↓              ↓
    ① ── C(4) ──→ ④ ── D(1) ──→ ⑤

求关键路径：
① etv正向推：事件最早发生时间
② ltv反向推：事件最晚发生时间
③ etv == ltv 的事件在关键路径上
```

### 4. C语言代码：关键路径概念

```c
// 关键路径
// etv: 正推，事件最早发生时间，取的是几条路之间的最大值，按拓扑排序的思想
// ltv: 倒推，事件的最晚发生时间，取的是几条路之间的最小值
// 关键路径：由etv和ltv相等的点构成的路径叫做关键路径
```

### 📌 易错点
| 易错点 | 说明 |
|--------|------|
| 拓扑排序需要**有向无环图** | 有环图不能拓扑排序 |
| **拓扑排序结果不唯一** | 多个入度为0顶点时可任意选择 |
| etv取最大值 | 前置活动都必须完成，取最长的 |
| ltv取最小值 | 后续活动的最晚开始时间，取最短的 |
