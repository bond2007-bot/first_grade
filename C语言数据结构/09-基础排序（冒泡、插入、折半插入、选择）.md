## 一、通用交换函数

```c
#include<stdio.h>

void swap(int* data, int m, int n)
{
    int temp;
    temp = data[m];
    data[m] = data[n];
    data[n] = temp;
}
```

## 二、冒泡排序（Bubble Sort）

**思想**：相邻元素两两比较，大的往后冒，每趟确定一个最大值放在末尾。

```c
void bubblesort(int* data, int len)
{
    for(int i = 0; i < len; i++)
    {
        for(int j = 0; j < len - i - 1; j++)
        {
            if(data[j] > data[j+1])
            {
                swap(data, j, j+1);
            }
        }
    }
}
```

### 执行过程示例
```
初始：47 35 60 95 77 15 20
第1趟：35 47 60 77 15 20 [95]   最大值95沉底
第2趟：35 47 60 15 20 [77 95]
第3趟：35 47 15 20 [60 77 95]
第4趟：35 15 20 [47 60 77 95]
第5趟：15 20 [35 47 60 77 95]
第6趟：15 [20 35 47 60 77 95]
```

### 特性
| 项目 | 值 |
|------|-----|
| 平均时间复杂度 | O(n²) |
| 最坏时间复杂度 | O(n²) |
| 最好时间复杂度 | O(n)（已有序） |
| 空间复杂度 | O(1) |
| 稳定性 | ✅ **稳定** |
| 适用场景 | 数据量小，要求算法稳定 |

## 三、直接插入排序（Insertion Sort）

**思想**：将每个数据和已经排好序的数据从后往前比较，找到合适位置插入。

```c
void insertsort(int* data, int len)
{
    int key;
    for(int i = 1; i < len; i++)  // 第一个数据相当于已插入
    {
        key = data[i];
        int j = i - 1;  // 从后往前比较
        while(j >= 0 && key < data[j])
        {
            data[j+1] = data[j];
            j--;
        }
        data[j+1] = key;  // 插入到正确位置
    }
}
```

### 执行过程示例
```
初始：47 35 60 95 77 15 20
i=1: key=35 → [35 47] 60 95 77 15 20
i=2: key=60 → [35 47 60] 95 77 15 20
i=3: key=95 → [35 47 60 95] 77 15 20
i=4: key=77 → [35 47 60 77 95] 15 20
i=5: key=15 → [15 35 47 60 77 95] 20
i=6: key=20 → [15 20 35 47 60 77 95]
```

### 特性
| 项目 | 值 |
|------|-----|
| 平均时间复杂度 | O(n²) |
| 最坏时间复杂度 | O(n²) |
| 最好时间复杂度 | O(n) |
| 空间复杂度 | O(1) |
| 稳定性 | ✅ **稳定** |
| 适用场景 | 数据量小或基本有序 |

## 四、折半插入排序（Binary Insertion Sort）

**思想**：在直接插入排序的基础上，用**折半查找**找到插入位置，减少比较次数。

```c
void BinaryInsertsort(int* data, int len)
{
    int low, high, mid;
    int key;

    for(int i = 0; i < len; i++)
    {
        key = data[i];
        low = 0;
        high = i - 1;

        // 折半查找key应该插入的位置（low的最终位置）
        while(low <= high)
        {
            mid = (low + high) / 2;
            if(key < data[mid])
                high = mid - 1;
            else
                low = mid + 1;
        }

        // 将low及以后的元素后移
        int j = i - 1;
        while(j >= low)
        {
            data[j+1] = data[j];
            j--;
        }
        data[low] = key;  // 插入
    }
}
```

### 特性
| 项目 | 值 |
|------|-----|
| 平均时间复杂度 | O(n²) |
| 最好时间复杂度 | O(n log n) |
| 最坏时间复杂度 | O(n²) |
| 空间复杂度 | O(1) |
| 稳定性 | ✅ **稳定**（注意：保证`key < data[mid]`时去左半） |
| 特点 | 减少了比较次数，但移动次数不变 |

## 五、简单选择排序（Selection Sort）

**思想**：在未排序的数据中一直找最小的，放到这一轮未排序的最前面。

```c
void selectsort(int* data, int len)
{
    int min;  // 记录最小值的下标
    for(int i = 0; i < len; i++)
    {
        min = i;
        for(int j = i; j < len; j++)
        {
            if(data[min] > data[j])
                min = j;
        }
        if(i != min)  // 最小不是起始位置才交换
            swap(data, i, min);
    }
}
```

### 执行过程示例
```
初始：47 35 60 95 77 15 20
第1趟：min=5(data[5]=15) → [15] 35 60 95 77 47 20
第2趟：min=6(data[6]=20) → [15 20] 60 95 77 47 35
第3趟：min=6(data[6]=35) → [15 20 35] 95 77 47 60
第4趟：min=6(data[6]=47) → [15 20 35 47] 77 95 60
第5趟：min=6(data[6]=60) → [15 20 35 47 60] 95 77
第6趟：min=7(data[7]=77) → [15 20 35 47 60 77] 95
第7趟：→ [15 20 35 47 60 77 95]
```

### 特性
| 项目 | 值 |
|------|-----|
| 时间复杂度 | O(n²)（最坏/平均/最好都一样） |
| 空间复杂度 | O(1) |
| 稳定性 | ❌ **不稳定** |
| 适用场景 | 小规模数据，简单直观 |

## 六、四种排序算法对比

| 排序算法 | 平均时间复杂度 | 最好 | 最坏 | 空间复杂度 | 稳定性 |
|---------|--------------|------|------|-----------|-------|
| **冒泡排序** | O(n²) | O(n) | O(n²) | O(1) | ✅ 稳定 |
| **直接插入排序** | O(n²) | O(n) | O(n²) | O(1) | ✅ 稳定 |
| **折半插入排序** | O(n²) | O(n log n) | O(n²) | O(1) | ✅ 稳定 |
| **简单选择排序** | O(n²) | O(n²) | O(n²) | O(1) | ❌ 不稳定 |

## 七、期末考试考点

### 📌 核心考点
1. **手动模拟各趟排序结果**（必考大题）
2. **算法稳定性判定**（选择题常考）
3. **时间/空间复杂度对比**
4. **算法适用场景选择**

### 📌 常见题型
- **选择题**：时间复杂度、稳定性、适用场景
- **填空题**：写出某趟排序后的序列
- **简答题**：比较不同排序算法的优劣
- **算法题**：排序算法代码填空或实现

### 📌 不稳定排序记忆口诀

> **"快些选堆"** —— 快速排序、希尔排序、简单选择排序、堆排序

### 📌 稳定排序
直接插入排序、冒泡排序、归并排序、基数排序

### 📌 易错点
- 冒泡排序内层循环范围是 `len-i-1`
- 插入排序从 i=1 开始（第一个元素视为已排序）
- 选择排序的比较次数与初始状态**无关**（始终是 n(n-1)/2 次）
