## 一、堆的概念

**堆**：基于完全二叉树结构的选择排序，利用堆来选出当前无序部分的最大或最小的元素。

### 大根堆（最大堆）
- **每个节点都大于等于其子节点**（上大下小）
- 堆顶元素是最大值
- 用于**升序排序**

### 小根堆（最小堆）
- **每个节点都小于等于其子节点**（上小下大）
- 堆顶元素是最小值
- 用于**降序排序**

```
大根堆示例：        小根堆示例：
      95                 15
     /  \               /  \
   77   60             28   35
  / \   / \           / \   / \
 47 35 15 28         47 77 60 95
```

### 堆的存储
- 用**数组**顺序存储
- 编号从1开始（下标0通常用作哨兵或占位）
- 编号关系：
  - 根节点为 `i`，左孩子为 `2*i`，右孩子为 `2*i + 1`
  - 父节点为 `i / 2`

## 二、堆排序代码实现

```c
#include<stdio.h>

void swap(int* data, int m, int n)
{
    int temp;
    temp = data[m];
    data[m] = data[n];
    data[n] = temp;
}

// 向下调整堆（从root开始）
void adjust(int* data, int root, int len)
{
    int child;
    int temp = data[root];
    child = 2 * root;  // 左孩子

    while(child <= len)
    {
        // 右孩子存在且更大，选右孩子
        if(child < len && data[child] < data[child+1])
            child++;

        if(temp > data[child])
            break;  // 当前节点比孩子大，无需调整

        // 子节点值更大，上移
        data[child/2] = data[child];
        child = child * 2;  // 继续向下调整
    }
    data[child/2] = temp;
}

void heapSort(int* data, int len)
{
    // 1. 建堆：从最后一个非叶子节点向前调整
    for(int i = len / 2; i > 0; i--)
    {
        adjust(data, i, len);
    }

    // 2. 排序：将堆顶（最大值）交换到数组末尾，调整剩余部分
    for(int i = len; i > 1; i--)
    {
        swap(data, 1, i);         // 堆顶与最后一个元素交换
        adjust(data, 1, i - 1);   // 调整剩余部分为大根堆
    }
}
```

### 测试代码

```c
int main()
{
    // data[0]不使用，仅用于占位
    int data[] = {-1, 47, 35, 60, 95, 77, 15, 28};
    int len = sizeof(data) / sizeof(data[0]);

    heapSort(data, len - 1);  // 不包含占位的data[0]

    for(int i = 1; i < len; i++)
    {
        printf("%d ", data[i]);
    }
    return 0;
}
```

## 三、堆排序执行过程详解

### 初始建堆过程

```
初始数组：[-1, 47, 35, 60, 95, 77, 15, 28]
（下标：   0   1   2   3   4   5   6   7）

初始完全二叉树：
        47
       /  \
      35   60
     / \   / \
    95 77 15 28

从最后一个非叶子节点开始调整（len/2=7/2=3，即节点60）：
① 调整节点60（下标3）：60>15, 60>28 ✓
② 调整节点35（下标2）：35<95 → 35和95交换
        47
       /  \
      95   60
     / \   / \
    35 77 15 28

③ 调整节点47（下标1）：47<95且47<60 → 选95上移
   47下移后与77比较 → 47<77 → 交换47和77
        95
       /  \
      77   60
     / \   / \
    35 47 15 28

建堆完成！堆顶为最大值95 ✓（大根堆）
```

### 排序过程

```
第1趟：交换data[1]=95和data[7]=28 → [28] 重新调整
调整：选77上移，28下沉...
得到：77 47 60 35 28 15 [95]

第2趟：交换data[1]=77和data[6]=15 → [15] 重新调整
得到：60 47 15 35 28 [77 95]

第3趟：60和28交换...
得到：47 35 28 15 [60 77 95]

...最终：[15 28 35 47 60 77 95]
```

## 四、特性总结

| 项目 | 值 |
|------|-----|
| 平均时间复杂度 | **O(n log n)** |
| 最坏时间复杂度 | **O(n log n)** ✅ |
| 最好时间复杂度 | O(n log n) |
| 空间复杂度 | O(1)（原地排序） |
| 稳定性 | ❌ **不稳定** |
| **建堆时间** | O(n) |
| 适用场景 | 取top k、优先级队列 |

### 与其他排序对比

| 场景 | 推荐算法 |
|------|---------|
| 只取前k个最大/最小 | **堆排序**（无需全排序） |
| 需要稳定的O(n log n) | 归并排序 |
| 平均最快 | 快速排序 |
| 优先级管理 | **堆**（优先队列） |

## 五、期末考试考点

### 📌 核心考点
1. **初始建堆过程**（必考大题）
2. **每趟排序后的状态**（必考大题）
3. **堆的定义和性质**
4. **堆的插入和删除操作**

### 📌 常见题型
- **选择题**：堆的定义、时间复杂度、是否为堆的判断
- **填空题**：给定序列构建堆、写出每趟排序后序列
- **简答题**：建堆过程描述、堆排序与其它排序对比
- **算法题**：堆调整函数（HeapAdjust）的填空或实现

### 📌 重要性质
1. **大根堆的堆顶是最大值**
2. **从最后一个非叶子节点（len/2）开始向前调整**
3. **完全二叉树的节点编号关系**：孩子 = 2*parent 和 2*parent+1
4. **堆排序不稳定**

### 📌 易错点
- `data[0]`通常不存数据（或存哨兵），有效数据从`data[1]`开始
- 调整时先比较左右孩子，再与父节点比较
- 建堆不是排序，建堆只是构建堆结构
- 堆排序需要通过不断交换堆顶到末尾来完成排序
