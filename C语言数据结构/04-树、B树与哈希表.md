## 一、B树（B-Tree）—— 平衡多叉树

B树是多路版本的平衡查找树，也叫"平衡多叉树"。B树允许每个节点有多个关键字和多个孩子。

### 1. m阶B树的定义

一个m阶B树满足以下性质：

1. **每个节点最多有m个孩子**
2. **每个非根非叶子节点至少有 ⌈m/2⌉ 个孩子**
3. **每个非叶节点有k个孩子，就一定有k-1个关键字**
   - 对于任意节点，若有k个关键字，要么没有孩子，要么有k+1个孩子
4. **关键字按照中序遍历时有序的**
5. 关键字在节点中是排好序的，孩子在关键字之间穿插，每个孩子对应一个区间

### 2. B树的特点

```
                    ┌───────────────┐
                    │   k1 │ k2 │ k3 │
                    └───┬───┴───┴───┬──┘
             ┌──────────┤           ├──────────┐
             ▼          ▼           ▼          ▼
          ┌─────┐   ┌─────┐    ┌─────┐    ┌─────┐
          │<k1  │   │k1~k2│    │k2~k3│    │>k3  │
          └─────┘   └─────┘    └─────┘    └─────┘
```

- **B树的查找**：中序遍历
- 所有叶子节点都在同一层（平衡的）

### 3. B+树的特点

- **所有关键字只出现在叶子节点中**
- **非叶子节点仅用于索引**（不存储数据指针）
- **所有叶子节点按照大小顺序通过指针连接**，形成有序链表
- 常用于数据库索引（如MySQL索引）

### 4. B树 vs B+树对比

| 特性 | B树 | B+树 |
|------|-----|------|
| 关键字存储 | 内部节点也存数据 | 仅叶子节点存数据 |
| 叶子节点链接 | 无 | 有（有序链表） |
| 查找效率 | 不稳定（可能在内部节点找到） | 稳定（一定到叶子） |
| 范围查询 | 需要中序遍历 | 链表直接遍历 |
| 应用 | 文件系统 | 数据库索引 |

## 二、散列表（Hash Table）—— 哈希表

### 1. 核心思想

用一个函数**直接计算**出数据存储的位置，不需要经过任何比较。

### 2. 相关概念

- **哈希函数**：将关键字映射到散列表中某个地址的函数
- **冲突（Collision）**：不同的关键字映射到同一个地址
- **装填因子（Load Factor）**：`α = 表中记录数 / 散列表长度`

### 3. 散列函数的构造方法

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **除留余数法** | `H(K) = K mod p`（p为不大于表长的质数） | 最常用 |
| **数字分析法** | 分析关键字分布，取分布均匀的几位 | 关键字已知且位数多 |
| **平方取中法** | 取关键字平方后的中间几位 | 不知道关键字分布 |

### 4. 处理冲突的方法

| 方法 | 描述 | 特点 |
|------|------|------|
| **开放地址法** | 从冲突位置开始往后找空闲位置 | 线性探测、二次探测 |
| **再散列法** | 使用另一个哈希函数计算新地址 | 需要多个哈希函数 |
| **链地址法（拉链法）** | 每个地址是一个链表，冲突的挂上去 | 最常用，查找稳定 |

### 5. 开放地址法详解

**线性探测法**：
```
H(K) = K mod p
Hᵢ = (H(K) + i) mod m   (i = 1, 2, 3, ...)
```

**二次探测法**：
```
Hᵢ = (H(K) ± i²) mod m   (i = 1, 2, 3, ...)
```

### 6. 链地址法（拉链法）

```
索引    链表
 0 ─→ [key1] → [key2]
 1 ─→ [key3]
 2 ─→ ∅
...
```

## 三、期末考试考点

### 📌 B树考点
1. **B树的定义和性质**（选择题/填空题）
2. **m阶B树的最大/最小关键字数计算**
3. **B树的插入操作（节点分裂过程）**
4. **B树的删除操作（节点合并过程）**
5. **B+树与B树的区别**

### 📌 散列表考点
1. **哈希函数构造**（除留余数法最常考）
2. **处理冲突方法**（线性探测、链地址法必考）
3. **平均查找长度（ASL）计算**：
   - 查找成功的ASL = Σ(比较次数 × 概率)
   - 查找失败的ASL
4. **装填因子对查找效率的影响**

### 📌 常见题型
- **选择题**：B树性质、哈希冲突处理方法
- **填空题**：B树阶数、关键字计算
- **简答题**：
  - 画出插入关键字后的B树
  - 给定哈希表操作，计算ASL
  - 画出链地址法处理冲突的哈希表
- **算法题**：哈希表查找算法

### 📌 平均查找长度（ASL）公式

```
ASL成功 = Σ(Pi × Ci)
Pi：第i个元素的查找概率
Ci：查找第i个元素需要比较的次数
```
